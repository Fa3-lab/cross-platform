What are Vendor Prefixes?

Browser-specific prefixes that allow experimental CSS features to be implemented

Browser Prefix Example Used For
WebKit (Chrome, Safari) -webkit- -webkit-transform Experimental features, older versions
Mozilla Firefox -moz- -moz-transform Firefox-specific implementations
Microsoft (IE, Edge) -ms- -ms-transform Internet Explorer and old Edge
Opera -o- -o-transform Old Opera versions (pre-WebKit)
⚠️ Important: Always include the standard property last! Browsers will use the last recognized property, so the standard version should override vendor-prefixed versions when supported.

Modern Tools & Best Practices

Automated solutions for handling vendor prefixes

💡 Modern Development: Don't write vendor prefixes manually! Use automated tools that handle this for you based on your browser support requirements.
/* Autoprefixer Configuration */ // package.json { "browserslist": [ "> 1%", "last 2 versions", "not ie <= 10" ] } // CSS Input (what you write): .box { display: flex; transform: rotate(45deg); background: linear-gradient(45deg, red, blue); } // CSS Output (what Autoprefixer generates): .box { display: -webkit-box; display: -ms-flexbox; display: flex; -webkit-transform: rotate(45deg); transform: rotate(45deg); background: -webkit-linear-gradient(45deg, red, blue); background: linear-gradient(45deg, red, blue); }

Popular Tools:

  • Autoprefixer: PostCSS plugin that adds vendor prefixes automatically
  • PostCSS: Tool for transforming CSS with JavaScript
  • Sass/SCSS: CSS preprocessor with mixin support for prefixes
  • Build tools: Webpack, Gulp, Parcel handle prefixing automatically
  • Can I Use: Website to check feature support across browsers
/* Sass Mixin for Manual Prefixing */ @mixin transform($property) { -webkit-transform: $property; -moz-transform: $property; -ms-transform: $property; -o-transform: $property; transform: $property; } @mixin transition($property) { -webkit-transition: $property; -moz-transition: $property; -o-transition: $property; transition: $property; } // Usage: .my-element { @include transform(rotate(45deg)); @include transition(all 0.3s ease); }

✅ Best Practices:

  • Use automated tools like Autoprefixer instead of manual prefixes
  • Define your browser support requirements clearly
  • Test on real devices and browsers, not just simulators
  • Keep your build tools and dependencies updated
  • Use feature detection where appropriate
  • Always put the standard property last
  • Remove unnecessary prefixes for features with good support
