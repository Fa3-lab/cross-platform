Image Optimization

Demonstrating lazy loading and responsive images

Lazy-loaded Images (scroll to see loading):

❌ Unoptimized

• Load all images immediately

• Large file sizes

• No compression

• Single resolution

Result: Slow initial load

✅ Optimized

• Lazy load images

• WebP format with fallbacks

• Proper compression

• Responsive images

Result: Fast initial load
/* Modern Image Optimization */ /* 1. Responsive Images with srcset */ <img src="image-400.jpg" srcset="image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w" sizes="(max-width: 400px) 400px, (max-width: 800px) 800px, 1200px" alt="Description"> /* 2. Modern format with fallbacks */ <picture> <source srcset="image.webp" type="image/webp"> <source srcset="image.avif" type="image/avif"> <img src="image.jpg" alt="Description"> </picture> /* 3. Lazy loading (native) */ <img src="image.jpg" loading="lazy" alt="Description">

/* Network-Aware Loading */ // 1. Detect connection quality const connection = navigator.connection || navigator.mozConnection || navigator.webkitConnection; if (connection) { console.log('Connection type:', connection.effectiveType); console.log('Downlink speed:', connection.downlink + ' Mbps'); console.log('RTT:', connection.rtt + 'ms'); } // 2. Adaptive loading based on connection function loadContent() { if (connection && connection.effectiveType === 'slow-2g') { // Load minimal content loadTextOnly(); } else if (connection && connection.effectiveType === '2g') { // Load low-res images loadLowResImages(); } else { // Load full content loadFullContent(); } } // 3. Save data preference if (connection && connection.saveData) { // User has enabled data saver loadMinimalContent(); }

Resource Management

Monitoring and optimizing resource usage

Click "Measure Performance" to see resource usage...
⚠️ Device Considerations:
  • Low-end mobile devices: Limited CPU, memory, and battery
  • Older browsers: May not support modern optimization APIs
  • Slow networks: Prioritize critical resources
  • Data plans: Users may have limited data allowances
/* Performance Monitoring */ // 1. Performance Observer API const observer = new PerformanceObserver((list) => { for (const entry of list.getEntries()) { console.log('Performance entry:', entry); } }); observer.observe({entryTypes: ['measure', 'navigation', 'resource']}); // 2. Core Web Vitals import {getLCP, getFID, getCLS} from 'web-vitals'; getLCP(console.log); // Largest Contentful Paint getFID(console.log); // First Input Delay getCLS(console.log); // Cumulative Layout Shift // 3. Memory usage (Chrome only) if ('memory' in performance) { console.log('Used:', performance.memory.usedJSHeapSize); console.log('Total:', performance.memory.totalJSHeapSize); console.log('Limit:', performance.memory.jsHeapSizeLimit); } // 4. Device capabilities console.log('Hardware concurrency:', navigator.hardwareConcurrency); console.log('Device memory:', navigator.deviceMemory + 'GB');
